This week was different.

Heavy rain meant I couldn't attend my Embedded Systems & AI classes. At first, I thought I would fall behind.

Instead, something unexpected happened.

Without classroom lectures, I started challenging myself to solve problems on my own. Rather than waiting for assignments, I began creating my own.

One challenge I gave myself was to simulate a **digital clock** in C—without using the `clock.h` library.

The goal wasn't just to make the clock work.

The goal was to build the logic myself.

I wanted to think through questions like:

*   How does time progress every second?
    
*   What happens when seconds reach 60?
    
*   How do minutes and hours update?
    
*   How do I handle the transition from 23:59:59 to 00:00:00?
    

Initially, it felt difficult.

But every time I solved one small problem, I understood the logic a little better.

## Building Logic Before Writing Code

Another project I'm currently working on is a simple **Tic-Tac-Toe game**.

At first glance, it sounds like a beginner's project.

But once I started building it, I realized how many concepts come together:

*   Functions
    
*   2D Arrays
    
*   Conditional Statements
    
*   Loops
    
*   Input Validation
    
*   Modular Programming
    

More importantly, it forced me to think before writing code.

Instead of asking, "How do I code this?", I started asking, "How should the game actually work?"

That small change in thinking made a huge difference.

## Functions Changed the Way I Write Programs

One thing has become very clear over the past few weeks.

Functions are probably one of the most important concepts I've learned so far.

Earlier, I used to think of programs as one long sequence of instructions.

Now I naturally start breaking problems into smaller responsibilities.

For my Tic-Tac-Toe project, instead of writing everything inside `main()`, I began creating functions such as:

*   Initialize the board
    
*   Display the board
    
*   Get player input
    
*   Switch players
    
*   Check if a move is valid
    

This made the program easier to understand, debug, and improve.

I can already see why modular programming is so important in embedded systems.

## Using AI Differently

One of the biggest changes in my learning journey has been how I use AI.

Earlier, I would often ask AI for the complete solution whenever I got stuck.

Now, I use AI differently.

I treat it as a mentor instead of a solution provider.

Rather than asking:

> "Write the code for me."

I ask questions like:

*   Is my logic correct?
    
*   Am I thinking about this problem the right way?
    
*   Can you point out where my approach breaks?
    
*   Help me understand the mistake without giving me the full solution.
    

This approach is slower.

Sometimes it is frustrating.

But I have learned far more by solving problems myself than by simply copying an answer.

Every mistake teaches me something that I am unlikely to forget.

## My Biggest Takeaway

This week didn't introduce a brand-new programming concept.

Instead, it taught me something even more valuable.

Learning programming isn't about memorizing syntax.

It's about developing the ability to think logically, break problems into smaller pieces, and improve through practice.

Rain may have cancelled my classes, but it didn't stop my learning.

If anything, it pushed me to become a more independent learner.

And I think that's one of the most valuable lessons I've learned so far.
